\begin{abstract}
We present the Temporal Gait Transformer (TGT), a classification model that takes raw smart insole sensor data and identifies five distinct walking activities: normal walking, injury walking, stepping, swaying, and jumping. The input data comes from pressure sensors, accelerometers, and orientation quaternions on both feet. After merging left and right insole streams into synchronised rows at \SI{60}{Hz}, we segment the recording into overlapping windows of 20 timesteps (approximately 0.33 seconds each). Each timestep carries 50 sensor features. The TGT projects every timestep into a 128 dimensional embedding, adds a fixed sinusoidal positional encoding, prepends a learnable classification token, and passes the full sequence through four Transformer encoder layers with multi head self attention. A final linear head maps the classification token to five class probabilities. The model is trained with cross entropy loss and the AdamW optimiser over 60 epochs with cosine annealing of the learning rate. We evaluate on a held out test split from the training subjects as well as a fully external validation set collected from a separate group of subjects.
\end{abstract}

\section{Data Preprocessing}

\subsection{Data Types}
Data is collected from smart insoles on both feet of each subject. At every time step, the insoles measure pressure at multiple locations and provide accelerometer, gyroscope, and magnetometer readings, along with a computed quaternion and an insole label (right or left). Redundant fields include internal time stamps for each sub-device and battery status. A detailed list of all per-timestep features is provided in Table~\ref{tab:insole-features} in the Appendix.

\subsection{Separating Input Data}
The raw data for each walking condition is first separated into right and left insole streams using the insole identifier $ele\_36$. We then compare the time stamps of rows from the right and left insoles and match them into a single combined row when the difference in recording times satisfies the constraint $\delta t < 0.02$\,s. Rows that do not satisfy this constraint are discarded. We remove redundant sub-time fields and drop gyroscope and magnetometer readings, because this information is already captured by the quaternions. The resulting dataset, with all retained channels, is summarised in Table~\ref{tab:insole-features-selected} in the Appendix. Each remaining row is a full entry for both feet, containing 50 features in total: 18 pressure values, 3 accelerometer axes, and 4 quaternion components per foot.

\subsection{Sampling Rate}
After merging, the data is resampled to a fixed \SI{60}{Hz}. If the merged stream has a frame rate below 60\,Hz we upsample by linear interpolation. If it is above 60\,Hz we downsample by selecting evenly spaced indices. This ensures that every recording, regardless of the original device clock, produces a uniform 60 samples per second stream.

\subsection{Windowing}
We define a window parameter $W$ as the number of consecutive rows provided to the model. In our configuration we use $W = 20$, which corresponds to an input segment of approximately 0.33 seconds. The stride between consecutive windows is set to $S = 5$, giving a 75\% overlap. This overlap produces more training samples and ensures that transient gait events are captured even if they fall near a window boundary. Each window is therefore a matrix of shape $20 \times 50$.

\subsection{Split}
The labelled windows are randomly split into three sets: 80\% for training, 10\% for validation, and 10\% for testing. The validation set is used during training to monitor generalisation and to select the best checkpoint. A separate external validation set was collected independently from Group~10 subjects who did not appear in the training data. This external set contains four of the five classes (Normal walking, Injury walking, Stepping, Swaying) and is used as an additional held out evaluation after training is complete.


\section{Methodology}

The architecture of the TGT follows a standard Transformer encoder pipeline adapted for time series classification. The complete flow is: linear projection, positional encoding, classification token, Transformer encoder stack, layer normalisation, and a linear classification head. We describe each stage below.

\subsection{Linear Projection to 128 Dimensions}

The input to the model is a window of shape $W \times 50$, where each of the $W = 20$ rows is a single timestep with 50 sensor features. For every timestep we apply a learned linear projection $\mathbf{z}_t = W_{\text{proj}}\,\mathbf{x}_t + \mathbf{b}_{\text{proj}}$ that maps the 50 dimensional input vector $\mathbf{x}_t$ to a 128 dimensional embedding $\mathbf{z}_t$. This projection is shared across all timesteps and is trained end to end with the rest of the model. After this step we have a $20 \times 128$ matrix of token embeddings.

\subsection{Positional Encoding}

Since the Transformer has no built in notion of ordering, we add a fixed positional encoding to each token embedding so the model can distinguish early timesteps from later ones within the window. For a given position $\text{pos} \in \{0, 1, \dots, W{-}1\}$ and a dimension index $i \in \{0, 1, \dots, 63\}$ we compute:

\begin{align}
PE_{\text{pos},\, 2i}   &= \sin\!\left(\frac{\text{pos}}{10000^{\,2i / d_{\text{model}}}}\right) \\[4pt]
PE_{\text{pos},\, 2i+1} &= \cos\!\left(\frac{\text{pos}}{10000^{\,2i / d_{\text{model}}}}\right)
\end{align}

with $d_{\text{model}} = 128$. The denominator $10000^{2i/d_{\text{model}}}$ means that lower dimensions oscillate rapidly and capture fine grained local timing, while higher dimensions oscillate slowly and encode a broader sense of where the timestep sits within the segment. The result is a $20 \times 128$ matrix of positional vectors which we add element wise to the projection matrix.

\subsection{Classification Token}

Before passing the sequence into the Transformer we prepend a special learnable vector called the [CLS] token. This token has the same dimensionality (128) as the timestep embeddings but is not tied to any specific timestep. It is a free parameter that the model learns during training. Its purpose is to serve as an aggregation point: after the Transformer processes the full sequence, the [CLS] token will have attended to every timestep and should contain a summary representation of the entire window. After prepending, the input to the Transformer is a matrix of shape $21 \times 128$ (20 timestep tokens plus 1 classification token).

\subsection{Transformer Encoder}

The core of the model is a stack of $N = 4$ identical Transformer encoder layers. Each layer consists of two sub-blocks.

\subsubsection{Multi Head Self Attention}

For each of the 21 tokens, the layer computes three vectors: a Query ($Q$), a Key ($K$), and a Value ($V$), each obtained by a learned linear projection. The attention score between any two tokens is the dot product of one token's query with another token's key, scaled by $\sqrt{d_{\text{model}}}$. These scores are collected into a $21 \times 21$ matrix and passed through a softmax so that each row sums to one. Multiplying this attention matrix by the Value matrix produces a new $21 \times 128$ matrix where each token is now a weighted combination of all other tokens, according to how relevant they are.

We use $H = 4$ attention heads in parallel. Each head operates on a $128/4 = 32$ dimensional slice of the embedding, computes its own attention independently, and the results are concatenated back to 128 dimensions.

\subsubsection{Feed Forward Network}

After attention, each token passes independently through a two layer feed forward network (FFN). The FFN first projects from 128 to 512 dimensions (a 4$\times$ expansion), applies a Gaussian Error Linear Unit (GELU) activation, and then projects back to 128 dimensions. Residual connections and layer normalisation are applied around both the attention sub-block and the FFN sub-block.

A dropout rate of 0.1 is applied inside both the attention and FFN sub-blocks. The four encoder layers are applied sequentially, and each one refines the token representations further.

\subsection{Layer Normalisation and Classification Head}

After the final encoder layer, we extract the [CLS] token (position 0 of the 21 token sequence). This single 128 dimensional vector is passed through a Layer Normalisation step and then through a fully connected linear layer $\mathbf{o} = W_{\text{cls}}\,\mathbf{h}_{\text{CLS}} + \mathbf{b}_{\text{cls}}$ that outputs 5 values, one for each class. A softmax is applied to these logits to obtain a probability distribution over the five activities.

\subsection{Training Configuration}

The model is trained with standard backpropagation using Cross Entropy Loss between the predicted class probabilities and the one hot encoded ground truth labels. We use the AdamW optimiser, which adds decoupled weight decay to the standard Adam update rule. The specific training parameters are:
